### 1、Redis是单线程的为什么还这么快

```
1、因为Redis把数据全部保存在内存中，访问和修改都无需磁盘I/O，速度接近于CPU的内存访问极限
2、Redis底层实现的数据结构是哈希表，哈希表的查询复杂度是O(1)的
3、多线程程序需要频繁的加锁来保证数据的一致性，还会带来上下文切换，锁竞争和死锁风险，使用单线程事件循环消除锁竞争，减少了CPU的上下文切换开销
4、使用 I/O 多路复用机制
5、Redis 命令是原子化、短小、无复杂逻辑的操作，每次执行时间极短
```

### 2、IO多路复用是什么

```
让一个线程同时高效地管理多个网络连接
操作系统提供了几种系统调用，让一个线程可以监听多个文件描述符（socket）
epoll：事件驱动、无上限、性能极高
while (true) {
    1️⃣ 监听所有客户端 socket（通过 epoll_wait）
    2️⃣ 获取就绪的事件列表（哪些连接有读/写事件）
    3️⃣ 逐个执行对应命令（读取请求 -> 执行 -> 写回结果）
}
  优势         说明               
  高并发        一个线程可同时管理成千上万个连接 
  低开销        无需创建/销毁线程、无锁竞争   
  响应快      	 即时响应事件，无需轮询或等待   
CPU 利用率高	 几乎所有时间都在做“有用的事”  
Redis、Nginx、Node.js 都选择了单线程 + I/O 多路复用的模型
换个通俗易懂的话就是，一个线程管理多个
一个线程，同时管理多个 I/O 对象（文件描述符、socket 连接）
```

### 3、如何实现分页

```
SELECT * FROM users
ORDER BY id
LIMIT 10 OFFSET 10;这是对应mysql的实现语句
SELECT * FROM users
WHERE id > last_id
ORDER BY id
LIMIT 10;
方法1：limit,offset
方法2：last_id,limit
```
